DBMS Keys

Introduction to Keys in DBMS

Keys are the unique identifiers used in DBMS to uniquely identify a tuple(s) or rows in a table. Understanding keys is crucial for database management and ensuring data integrity.

Types of Keys

Primary Key

A primary key is a field (or a combination of fields) that uniquely identifies each record in a table. It ensures that every record is unique and not null.

Example: In the Students table, ID is the primary key.

ID Name Address Course RegistrationNumber Email
1 Rakesh Sharma Kathmandu Computer Science 112233 rakesh@example.com
2 Aakash Sherpa Pokhara Electronics 223344 aakash@example.com
3 Sital Thapa Bhaktapur Mechanical 334455 sital@example.com
4 Krishna Bhusal Lalitpur Civil 445566 krishna@example.com

Why It Matters: The ID must be unique for each student, ensuring no two students have the same ID and every student can be uniquely identified.

Foreign Key

A foreign key is a field (or a combination of fields) in one table that refers to the primary key in another table. It creates a relationship between the two tables.

Examples:

  • In the Enrollments table, StudentID is a foreign key that refers to ID in the Students table.
  • In the Enrollments table, CourseID is a foreign key that refers to CourseID in the Courses table.

Why It Matters: It links Enrollments with Students and Courses, ensuring that every StudentID and CourseID in Enrollments corresponds to valid entries in Students and Courses, respectively.

Composite Key

A composite key uses two or more columns to uniquely identify a record in a table.

Example: In the Enrollments table, StudentID and CourseID together form the composite key.

Why It Matters: The combination of StudentID and CourseID uniquely identifies each enrollment record, ensuring no duplication.

Alternate Key

An alternate key is any candidate key that is not chosen as the primary key. These are unique identifiers that can also be used to access records.

Example: In the Students table, RegistrationNumber could serve as an alternate key.

ID Name Address Course RegistrationNumber Email
1 Rakesh Sharma Kathmandu Computer Science 112233 rakesh@example.com
2 Aakash Sherpa Pokhara Electronics 223344 aakash@example.com
3 Sital Thapa Bhaktapur Mechanical 334455 sital@example.com
4 Krishna Bhusal Lalitpur Civil 445566 krishna@example.com

Why It Matters: RegistrationNumber is unique for each student and can be used to identify students, ensuring data integrity.

Candidate Key

Candidate keys are the set of fields from which the primary key is selected. Each candidate key can uniquely identify records in a table.

Example: In the Students table, both ID and RegistrationNumber are candidate keys.

ID Name Address Course RegistrationNumber Email
1 Rakesh Sharma Kathmandu Computer Science 112233 rakesh@example.com
2 Aakash Sherpa Pokhara Electronics 223344 aakash@example.com
3 Sital Thapa Bhaktapur Mechanical 334455 sital@example.com
4 Krishna Bhusal Lalitpur Civil 445566 krishna@example.com

Why It Matters: Candidate keys are essential in selecting the best primary key that ensures unique identification of records.

Super Key

A super key is a set of one or more columns that can be used to uniquely identify a record in a table. Every primary key is a super key, but not every super key is a primary key.

Example: In the Students table, ID, RegistrationNumber, and Email together can be a super key.

ID Name Address Course RegistrationNumber Email
1 Rakesh Sharma Kathmandu Computer Science 112233 rakesh@example.com
2 Aakash Sherpa Pokhara Electronics 223344 aakash@example.com
3 Sital Thapa Bhaktapur Mechanical 334455 sital@example.com
4 Krishna Bhusal Lalitpur Civil 445566 krishna@example.com

Why It Matters: Super keys help in understanding all possible ways to identify records uniquely and assist in database normalization.

Unique Key

A unique key is a set of fields in a table that must hold unique values. Unlike primary keys, unique keys can have null values.

Example: In the Students table, Email can be a unique key.

ID Name Address Course RegistrationNumber Email
1 Rakesh Sharma Kathmandu Computer Science 112233 rakesh@example.com
2 Aakash Sherpa Pokhara Electronics 223344 aakash@example.com
3 Sital Thapa Bhaktapur Mechanical 334455 sital@example.com
4 Krishna Bhusal Lalitpur Civil 445566 krishna@example.com

Why It Matters: Unique keys help in enforcing uniqueness for specific columns where unique identification is necessary but without the restriction of being a primary key.
